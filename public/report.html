Redovisning

Här kommer mina redovisningstexter från de olika kursmomenten i denna kursen.

kmom01

Min utvecklingsmiljö för denna kurs består av Visual Studio Code som huvudsaklig editor, Git för versionshantering och Node.js för att köra utvecklingsverktyg som ESLint och http-server. Jag har också installerat npm-paket som är nödvändiga för kursen enligt instruktionerna. Utvecklingsmiljön känns bekant och stabil, vilket gör det enkelt att fokusera på själva kodandet istället för att krångla med verktyg.

Installationen av kursens utvecklingsmiljö gick relativt smidigt. Jag hade redan Git och Node.js installerat sedan tidigare, så det var mest en fråga om att klona kursrepot och installera npm-dependencies. ESLint-konfigurationen var lite ny för mig, men efter att ha läst dokumentationen förstod jag hur den fungerar och varför den är användbar för att hålla konsistent kodkvalitet.

Jag har jobbat en del med terminalen tidigare, framförallt grundläggande Unix-kommandon som ls, cd, mkdir och git-kommandon. Det känns naturligt att navigera i filsystemet via kommandoraden och att använda Git för versionshantering. Terminalen är definitivt ett kraftfullt verktyg när man väl lärt sig grunderna.

HTML, CSS och JavaScript är tekniker jag stött på tidigare, men mest på en grundläggande nivå. Jag har byggt enkla webbsidor tidigare, men inte med den struktur och organisation som vi använder i denna kurs. Det var intressant att se hur man kan strukturera ett projekt med separata mappar för CSS och JavaScript, och hur man kan skapa en konsistent design med återanvändbara komponenter som header och footer.

Att komma igång med kursmomentet var överlag ganska rakt fram. Den stegvisa övningen hjälpte mycket att förstå hur alla delar hänger ihop. Det som var lite utmanande var att förstå CSS-layouten med grid för två-kolumners design, men efter lite experimenterande började det klarna. JavaScript-delen för att sätta aktuellt år var enkel men smart - det visar hur man kan använda JavaScript för praktiska funktioner.

Min TIL för detta kursmoment är hur kraftfullt aside och main är för att skapa layouter. Tidigare har jag mest använt divs för positionering

kmom02

Att jobba med responsiv design var både utmanande och lärorikt. Övningen gav mig en bra grund för att förstå media queries och flexbox, vilket var essentiellt för att lösa uppgiften. Jag fastnade initialt på hamburgermenyn för mobila enheter, men efter att ha studerat exemplen och dokumentationen fick jag det att fungera som tänkt.

Jag har faktiskt noterat responsiv design tidigare som användare, särskilt hur webbplatser anpassar sig när man växlar mellan desktop och mobil. Men att nu förstå mekanismen bakom det, med breakpoints och media queries, ger ett helt nytt perspektiv. Det är fascinerande hur man kan bygga en enda webbplats som fungerar sömlöst på olika enheter.

Med webbplatsens utseende är jag överlag nöjd, särskilt med hur navigeringen fungerar responsivt. Om jag hade haft mer tid skulle jag velat finslipa på övergångarna mellan olika skärmstorlekar och kanske lagt till några subtila animationer för att göra användarupplevelsen ännu smidigare.

JavaScript har varit en positiv överraskning. Efter att ha jobbat med if-satser och switch/case börjar jag se potentialen i språket. Det känns mindre komplext än jag först trodde, och möjligheten att direkt se resultatet i webbläsaren gör det roligt att experimentera med koden. Laborationen med kontrollstrukturer gav bra hands-on erfarenhet.

Min TIL (Today I Learned) för detta kursmoment är kraften i CSS flexbox tillsammans med media queries. Det är fascinerande hur några rader CSS kan transformera en layout från desktop till mobil på ett så elegant sätt.

kmom03

För min One-page webbplats utvecklade jag "TechFlow", ett företag specialiserat på innovativa tekniska lösningar. Färgpaletten bygger på en modern blå basfärg (#2563eb) som huvudfärg, kompletterad med en grön sekundärfärg (#10b981) och accentfärger i orange. Jag valde denna kombination för att förmedla både professionalitet (blå) och innovation (grön), medan den orange accentfärgen adderar energi och uppmärksammar viktiga element.

För att bygga webbplatsen använde jag mig av moderna ikoner och bilder som representerar teknologi och innovation. Bilderna valde jag noggrant för att matcha företagets professionella image. Texten formades genom att studera ledande tech-företags kommunikation, med fokus på att hålla språket tydligt och engagerande. Min inspiration kom från företag som erbjuder liknande tekniska tjänster, särskilt deras sätt att presentera komplexa lösningar på ett tillgängligt sätt.

Den största utmaningen var implementeringen av den responsiva navigeringsmenyn. Att få hamburgermenyn att fungera smidigt på mobila enheter och samtidigt behålla en elegant övergång till desktop-vyn krävde flera iterationer. En annan utmaning var att balansera innehållet i de olika sektionerna så att varje del fick tillräckligt med utrymme utan att överväldiga besökaren.

Till någon som är ny inom CSS skulle jag rekommendera att börja med att bemästra box-modellen - det är grunden för all layout. Fokusera sedan på att förstå flexbox, det är ett otroligt kraftfullt verktyg för layout. Ett tips är att aktivt använda webbläsarens utvecklarverktyg (devtools) - det ger direkt feedback och hjälper enormt när man experimenterar med olika CSS-egenskaper.

Min TIL för detta kursmoment är styrkan i att använda CSS-variabler för färger och återkommande värden. Genom att definiera färger som variabler i :root kunde jag enkelt uppdatera hela webbplatsens färgschema från ett enda ställe, vilket gjorde det mycket effektivare att experimentera med olika designalternativ.

kmom04

Detta kursmoment har varit särskilt lärorikt när det gäller att förstå samspelet mellan HTML, CSS och JavaScript. Genom att arbeta med DOM-manipulation har jag fått en djupare förståelse för hur dessa tre teknologier samverkar. JavaScript fungerar som en brygga mellan HTML-strukturen och CSS-stylingen, där vi kan dynamiskt modifiera både innehåll och utseende baserat på användarinteraktioner.

I uppgiften med DOM-manipulation blev det tydligt hur kraftfullt JavaScript är för att skapa interaktiva webbsidor. Att kunna välja element med document.getElementById och getElementsByClassName, för att sedan modifiera deras egenskaper och stil, öppnar upp för många möjligheter. Särskilt intressant var implementationen av tangentbordsstyrning för att flytta och manipulera boxar, där jag fick experimentera med både eventlyssnare och stilmanipulation.

När det gäller kodstrukturen strävade jag efter att hålla den så ren och modulär som möjligt. Jag skapade separata funktioner för olika funktionaliteter, som till exempel 'toggleSelected' för att hantera markering av boxar och 'onOutOfBounds' för att kontrollera positionering. Detta gjorde koden mer läsbar och underhållbar. En utmaning var att hantera alla olika events och se till att de fungerade korrekt tillsammans, särskilt när det gällde att kombinera musinteraktioner med tangentbordskommandon.

Min TIL för detta kursmoment är hur effektivt man kan använda classList för att hantera CSS-klasser i JavaScript. Att kunna toggla klasser som 'selected' och 'circular' istället för att direkt manipulera style-attribut ger en mycket renare och mer underhållbar kod. Det visade också tydligt fördelarna med att separera presentationslogik (CSS) från beteendelogik (JavaScript).

kmom05

Att arbeta med fetch-uppgiften var både utmanande och lärorikt. Implementationen av CO2-emissions jämförelsen mellan de nordiska länderna gav mig en djup förståelse för hur man kan hantera extern data i webbapplikationer. Det var särskilt intressant att se hur man kan kombinera async/await med felhantering för att skapa en robust applikation som ger användaren bra återkoppling.

Fördelarna med async/await blev väldigt tydliga under detta projekt. Jämfört med callbacks eller promises gjorde async/await koden mycket mer läsbar och lättare att följa. Det känns som att koden blir mer "linjär" och naturlig att läsa, särskilt när man behöver hantera flera sekventiella API-anrop. Felhantering med try/catch blev också mycket mer intuitiv jämfört med .catch() i promise-chains.

Den mest utmanande delen var att implementera logiken för att jämföra och markera högsta/lägsta värdena i tabellen. Att dynamiskt uppdatera tabellcellerna och lägga till rätt CSS-klasser krävde noggrann planering för att hålla koden modulär och underhållbar. Att hantera fallen där data saknades eller var ofullständig var också en intressant utmaning som krävde robust felhantering.

Min TIL för detta kursmoment är vikten av att ge användaren tydlig feedback under asynkrona operationer. Genom att implementera loading-states och felmeddelanden blev användarupplevelsen mycket bättre. Det var också lärorikt att se hur man kan strukturera asynkron kod på ett sätt som gör den både effektiv och lättförståelig.

kmom10

Krav 1: Landningssida för spelet

För projektet skapade jag en komplett onepage i public/project.html som marknadsför Duckhunt 2. Sidan är uppbyggd i sektioner: en hero-sektion med animerad emoji och call-to-action-knapp, en tvåkolumnslayout som beskriver spelidén, en trekolumnslayout med feature-kort, en spel-sektion där användaren kan välja svårighetsgrad direkt på onepagen, och en highscore-sektion som förklarar funktionen. Footer är indelad i tre kolumner med spelinformation, snabblänkar och kontaktinfo. Allt är stilat i ett grönt tema med CSS-variabler och sidan är responsiv via media queries för 768px och 600px. JavaScript i project.js hanterar hamburgermenyn, smooth scrolling, aktiv nav-highlighting via IntersectionObserver och header-scroll-effekten.

Krav 2: Spelet Duckhunt, version 2

Jag vidareutvecklade spelet från kmom06 med flera förbättringar. Det viktigaste kravet var att flytta ut konfigurationen till tre separata JSON-filer: easy.json, normal.json och hard.json. Varje fil innehåller inställningar för speltid, hastighet, ankstorlek och bakgrundsbild. När användaren väljer svårighetsgrad i formuläret på duckhunt.html hämtas rätt fil med fetch() och konfigurationen tillämpas dynamiskt. Svårighetsgradsformuläret ersatte de gamla numeriska inputfälten med väldesignade radioknappar. Ankans storlek varierar nu per svårighetsgrad — 130px på lätt, 100px normalt och 70px på svår. Onepagen har dessutom en svårighetsväljarelista som skickar vald nivå via URL-parameter (?difficulty=easy) till spelet, som sedan automatiskt väljer rätt radioknapp.

Krav 3: Highscorelista i tabell

Highscore-systemet är implementerat i main.js som ett in-memory-array (highscores) som nollställs automatiskt när sidan laddas om — precis som kravet specificerar. Efter varje spelomgång sparas spelarens poäng, vald svårighetsgrad och tidsstämpel i arrayen. Arrayen sorteras automatiskt fallande efter poäng. När spelet är slut visas en game-over-overlay med spelarens slutpoäng och hela highscore-listan i en stylad tabell med kolumnerna: plac, poäng, nivå och tid. Den senaste omgångens resultat markeras med gult i tabellen. Under spelet visas poäng och nedräkningstimer tydligt på skärmen i en stilad poängruta. En sektion i onepagen förklarar highscore-featuren för användaren.

Allmänt om projektet

Projektet gick överlag bra att genomföra. Det som tog mest tid var att refaktorera spelet från hårdkodade värden till ett system som dynamiskt hämtar konfiguration via fetch och JSON-filer — det krävde att man tänkte igenom hur data flödar genom applikationen. Onepagen var relativt enkel att bygga eftersom jag hade kmom03 som mall, men att finslipa CSS, responsivitet och JavaScript-interaktivitet tog sin tid. Det roligaste var att implementera highscore-systemet eftersom det kopplade ihop spelets logik med en konkret visuell presentation. Det svåraste var att hantera ESLint CSS-reglerna, som inte accepterade background som kortform med var()-värden. Projektet känns som ett rimligt och välstrukturerat avslut på kursen — det täcker HTML, CSS och JavaScript på ett balanserat sätt.

TIL för kmom10

Min TIL för detta kursmoment är att JSON-konfigurationsfiler via fetch är ett otroligt smidigt mönster för att göra en applikation konfigurerbar utan att röra källkoden. Att separera data från logik gör koden mer flexibel och lättare att underhålla. En enkel ändring i en JSON-fil kan förändra hela spelupplevelsen utan att man behöver peta i JavaScript-koden.

TIL för kursen som helhet

Det övergripande jag tar med mig från kursen är hur HTML, CSS och JavaScript samverkar som ett ekosystem. Varje teknik har sitt ansvarsområde: HTML för struktur, CSS för presentation och JavaScript för beteende. Att ha jobbat igenom hela kedjan — från statiska sidor via responsiv design och DOM-manipulation till asynkrona API-anrop och spelprogrammering — ger en solid och bred grund att stå på inför vidare studier och projekt.

Tankar om kursen (valfritt)

Kursen har en bra progression från enkla HTML-sidor hela vägen till ett interaktivt spel med asynkron kommunikation. Materialet är välstrukturerat och laborationerna är genomtänkta. Framöver skulle det vara intressant med ett kursmoment som introducerar ett enkelt komponentbaserat tänk, till exempel hur man kan återanvända HTML-strukturer med JavaScript-mallar. Det hade gett en naturlig brygga till moderna frontend-ramverk. Sammantaget gav kursen mig ett genuint intresse för webbutveckling. Betyg: 8/10 — jag rekommenderar kursen till alla som vill lära sig webbutveckling från grunden på ett strukturerat sätt.
